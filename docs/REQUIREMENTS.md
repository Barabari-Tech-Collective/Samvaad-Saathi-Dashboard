Dashboard for tracking ANALYTICS
Tracking at 4 levels:
Student level (micro)

Interview/session level

Segment level (role, difficulty, college)

System/product level (what’s working vs not)

Below is a complete analytics blueprint—no fluff.

1. STUDENT LEVEL ANALYTICS (Most Important)
   This tells you:
   👉 “Is this student improving or stuck?”
   Core Metrics
   A. Performance Tracking
   Latest score (speech + knowledge)

Average score (last 3 interviews)

Best score

Improvement rate (trend)

B. Skill Breakdown (Critical)
Track per student:
Speech:

WPM

Pause score

Filler density

Energy

Consistency

Knowledge:

Technical accuracy

Structure quality

Relevance
Examples given or not

👉 Store per metric history, not just final score.

C. Consistency Score
Variance across interviews

Are they stable or fluctuating?

D. Weak Area Detection (Auto-tagging)
For each student:
“High filler usage”

“Low energy”

“Poor structure”

“Too short answers”

👉 This becomes your recommendation engine backbone

E. Practice Compliance
Recommended exercises vs completed

Improvement after practice

👉 This is where real learning happens

F. Attempt Behavior
Number of interviews attempted

Reattempt frequency

Time between attempts

2. INTERVIEW / SESSION LEVEL
   This tells you:
   👉 “What happened in this specific interview?”
   Track per interview:
   Role selected

Technical

Score:

Performance drop off at question level and

3. SEGMENT LEVEL ANALYTICS
   Now zoom out.

A. Role-Level Analytics
👉 “Which roles are hardest?”
Track:
Avg score per role

Drop-off rate per role

Common weaknesses per role

Time spent per role

Example insight:
“Backend role → low structure scores”

B. Difficulty-Level Analytics
👉 “Are your difficulty levels actually meaningful?”
Track:
Avg score per level

Completion rate

Retry rate

If:
Easy = same as Medium → your system is broken

C. College-Level Analytics (Your BIG opportunity)
👉 “Which colleges need help the most?”
Track:
Avg score per college

Improvement rate per college

Usage frequency

Completion rate

Key Insights:
Which colleges improve fastest?

Which are stuck?

Which need intervention?

D. Underserved Segment Tracking
You said underserved communities → track:
First attempt vs latest score gap

Practice engagement

Drop-off points

👉 This is impact measurement

4. PRODUCT / SYSTEM ANALYTICS
   This tells you:
   👉 “Is your app actually working?”

A. Funnel Tracking
Users:
Sign up

Select role

Start interview

Complete interview

View report

Do practice

Track drop-off at each stage.

B. Report Usage
% users who open report

Time spent on report

Clicks on recommendations

C. Practice Effectiveness
This is HUGE:
Did users improve after practice?

Which exercises lead to score increase?

👉 Kill useless exercises

D. Retry Behavior
How many retries before improvement?

Do users spam retries without improving?

E. Question Effectiveness
Which questions:

cause low scores

cause drop-offs

👉 Improve or replace them

5. SCORING ANALYTICS
   You need to validate your system.
   Track:
   Correlation:

Speech score vs knowledge score

Score distribution:

Are all users getting 70–80? → bad system

6. ALERTS YOU SHOULD SET
   Don’t just store data—act on it.
   Student Alerts:
   No improvement after 3 attempts

High filler + low energy combo

Too many retries without progress

System Alerts:
Sudden drop in scores (model issue)

High failure rate in one role

One college performing extremely low

8. MOST IMPORTANT METRIC (Don’t Miss This)
   👉 Improvement Rate
   Formula:
   improvement = (latest score - first score)
   Track:
   per student

per college

per role

This is your true success metric.

9. What You’re Likely Missing (Critical)
1. Learning Loop Tracking
   User → report → practice → improvement
   If you don’t track this, your app is just analysis, not training.

1. Behavioral Patterns
   Example:
   fast speakers → also high fillers?

low energy → low knowledge scores?

This gives deep insights.

3. Engagement vs Performance
   Some users:
   practice a lot but don’t improve

→ your system is failing them

Brutal Truth
If you only track:
❌ scores
❌ usage
You’ll build a dashboard.
If you track:
✔ improvement
✔ behavior
✔ intervention success
You’ll build a learning system.
